# Contributing to CONCAP

**Control de Captación** · Intelligent Water Intake Monitoring and Automation System

</div>

---

> [!IMPORTANT]
> **Contributions are currently closed.**
> This repository is a technical showcase, not an open-source project. It contains no source code, and there is nothing to contribute to in the conventional sense.

<br/>

## Why contributions are closed

CONCAP is a **deployed commercial engineering system** operating on real water infrastructure serving a rural community. The repository exists to document its architecture and results — not to host collaborative development.

Three reasons drive this policy:

| Reason | Explanation |
|:---|:---|
| **Intellectual property** | Firmware, hardware design, and control algorithms are proprietary and withheld. There is no codebase in this repository to modify. |
| **Operational safety** | The system governs the water intake of a live aqueduct. Changes to a deployed installation follow a controlled engineering and commissioning process, not a pull request workflow. |
| **Scope** | This repository documents one specific installation. It is not a general-purpose framework intended for reuse or adaptation. |

<br/>

## What this means in practice

| Action | Status |
|:---|:---:|
| Pull requests | ❌ Not accepted — will be closed without review |
| Feature requests | ❌ Not accepted |
| Issues | ❌ Disabled |
| Forking to develop derivative work | ❌ Prohibited by [LICENSE](LICENSE) |
| Reading, studying, and evaluating the documentation | ✅ Encouraged |

<br/>

## What *is* welcome

Although code contributions are closed, **technical dialogue is not**. Reach out directly if you fall into any of these categories:

- 🔍 **Recruiters and hiring teams** evaluating embedded systems, IoT, or full-stack hardware engineering experience.
- 🤝 **Clients and organizations** interested in deploying a similar system for an aqueduct, treatment plant, or industrial water process.
- 🎓 **Engineers and researchers** who want to discuss the architecture, the ESP-NOW field link, the analog acquisition strategy, or the solar energy budget.
- 🏘 **Community aqueduct operators** facing the same turbidity problem and wanting to understand whether this approach fits their site.
- 📄 **Documentation corrections** — if you spot a factual error, a broken link, or an inconsistency in these documents, mention it by email.

Contact details are in the [Contact section of the README](README.md#-contact).

<br/>

## Security

If you believe you have found a security-relevant issue in anything described here, **do not open a public issue**. Follow the private disclosure process in [SECURITY.md](SECURITY.md).
